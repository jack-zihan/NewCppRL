## 概述

本文档详细分析了SAC（Soft Actor-Critic）算法在连续控制任务中的网络架构实现。该实现采用了深度卷积神经网络（CNN）与多层感知机（MLP）的混合架构，专门为处理视觉输入（128×128像素图像）和连续动作空间设计。

### 主要特点
- **输入模态**：视觉观测（RGB图像）+ 向量特征（状态信息）
- **输出空间**：连续动作空间（2维）
- **网络类型**：Actor-Critic架构
- **激活函数**：SiLU (Swish)激活函数
- **分布建模**：TanhNormal分布用于连续动作

## Policy Network (Actor) 详细结构

### 1. 整体架构
Policy网络负责学习策略π(a|s)，输出动作的概率分布参数。

```python
Policy Network 结构:
├── CNN编码器 (ConvEncoder)
│   ├── 卷积块1 (_ConvNetBlock)
│   ├── 卷积块2 (_ConvNetBlock)
│   └── 卷积块3 (_ConvNetBlock)
├── 特征融合层
│   └── Linear(CNN特征 + vector → 512)
├── MLP头部
│   └── Linear(512 → 512)
└── 动作分布参数输出
    ├── loc (均值): Linear(512 → 2)
    └── scale (标准差): Linear(512 → 2) + Softplus
```

### 2. 卷积块详细结构 (_ConvNetBlock)

每个卷积块包含：
```
_ConvNetBlock:
├── Conv2d层
├── MaxPool2d层 (stride=2)
├── ResNet块1
│   ├── SiLU激活
│   ├── Conv2d(3×3, padding=1)
│   ├── SiLU激活
│   └── Conv2d(3×3, padding=1)
└── ResNet块2 (与ResNet块1相同)
```

### 3. 具体参数配置

| 层级 | 类型 | 输入通道 | 输出通道 | 卷积核 | 步长 | 池化步长 |
|------|------|----------|----------|--------|------|----------|
| Block 1 | ConvNet | C_in | 32 | 3×3 | 1 | 2 |
| Block 2 | ConvNet | 32 | 64 | 3×3 | 1 | 2 |
| Block 3 | ConvNet | 64 | 64 | 3×3 | 1 | 2 |

## Q-Value Network (Critic) 详细结构

### 1. 整体架构
Q网络负责评估状态-动作对的价值Q(s,a)。

```python
Q-Value Network 结构:
├── CNN编码器 (与Policy网络共享架构)
│   ├── 卷积块1 (32通道)
│   ├── 卷积块2 (64通道)
│   └── 卷积块3 (64通道)
├── 特征融合层
│   └── Linear(CNN特征 + vector + action → 512)
├── MLP头部
│   └── Linear(512 → 512)
└── Q值输出
    └── Linear(512 → 1)
```

### 2. 关键差异
- **输入差异**：Q网络额外接收动作输入（2维）
- **向量维度**：Policy网络vec_dim=1，Q网络vec_dim=3（1个状态+2个动作）
- **输出差异**：Policy输出4维（2个均值+2个标准差），Q网络输出1维（Q值）

## Tensor维度变化追踪

### Policy Network 数据流

```
输入:
├── observation: [B, C, 128, 128]  # C为通道数（环境相关）
└── vector: [B, 1]

CNN处理流程:
[B, C, 128, 128] → Conv1+Pool → [B, 32, 64, 64]
                 → ResNet×2
[B, 32, 64, 64]  → Conv2+Pool → [B, 64, 32, 32]
                 → ResNet×2
[B, 64, 32, 32]  → Conv3+Pool → [B, 64, 16, 16]
                 → ResNet×2
[B, 64, 16, 16]  → Flatten    → [B, 64×16×16] = [B, 16384]

特征融合:
[B, 16384] + [B, 1] → [B, 16385] → Linear → [B, 512]

输出:
[B, 512] → Linear → [B, 512] → Linear → [B, 4]
         → NormalParamExtractor → loc: [B, 2], scale: [B, 2]
```

### Q-Value Network 数据流

```
输入:
├── observation: [B, C, 128, 128]
├── vector: [B, 1]
└── action: [B, 2]

CNN处理流程:
（与Policy Network相同）
[B, C, 128, 128] → ... → [B, 16384]

特征融合:
[B, 16384] + [B, 1] + [B, 2] → [B, 16387] → Linear → [B, 512]

输出:
[B, 512] → Linear → [B, 512] → Linear → [B, 1]
```

## 关键技术特性

### 1. 激活函数选择
- **SiLU (Swish)**：x * sigmoid(x)
  - 平滑、非单调
  - 提供更好的梯度流
  - 在深层网络中表现优异

### 2. 动作分布建模
- **TanhNormal分布**
  - 支持有界连续动作空间
  - 通过tanh变换将无界高斯映射到[-1, 1]
  - 提供重参数化技巧支持

### 3. ResNet连接
- **残差块设计**
  - 缓解梯度消失问题
  - 加速训练收敛
  - 每个卷积块包含2个ResNet块

### 4. 参数初始化
- **NormalParamExtractor**
  - scale_mapping: "biased_softplus_1.0"
  - scale_lb: 1e-4（防止标准差过小）

## 架构优势与设计理念

### 1. 模块化设计
- **组件解耦**：CNN编码器、MLP头部、分布参数化分离
- **复用性高**：Policy和Q网络共享CNN架构
- **易于扩展**：支持不同环境版本(v1-v5)的自动适配

### 2. 高效特征提取
- **层次化特征**：通过多层卷积逐步抽象
- **多尺度感知**：3次下采样获得不同尺度特征
- **特征融合**：视觉特征与向量特征有效结合

### 3. 稳定性设计
- **ResNet稳定训练**：避免深层网络训练困难
- **参数约束**：标准差下界防止数值不稳定
- **归一化输出**：TanhNormal确保动作在合理范围

### 4. 计算效率
- **参数共享**：减少总参数量
- **高效激活**：SiLU相比ReLU提供更好性能
- **合理深度**：3层卷积块平衡性能与效率

### 5. 环境适应性
- **自动规格检测**：根据环境自动配置输入输出维度
- **动作空间验证**：确保使用连续动作空间
- **多版本支持**：兼容NewPasture环境的v1-v5版本

## 总结

该SAC网络架构通过精心设计的CNN-MLP混合结构，有效处理高维视觉输入并生成连续动作。关键创新包括：

1. **深度卷积特征提取**：3层卷积块+ResNet结构
2. **多模态融合**：视觉与向量特征的有效结合
3. **稳定的分布建模**：TanhNormal分布处理有界动作
4. **现代激活函数**：SiLU提供更好的训练动态
5. **模块化架构**：高复用性和可扩展性

该设计在保持计算效率的同时，提供了强大的表征学习能力，特别适合机器人导航等需要视觉感知的连续控制任务。
