# 人类意图场（HIF）方向一致性与 v5 正则项实现核查报告（ultrathink）

## 一、结论摘要
- 数据格式：两套 HIF 生成管线（/home/lzh/HIFS 纹理提取 与 envs_new/tools 形状/性质提取）输出完全一致：二维 `float32`，背景为 `-1.0`，前景角度 ∈ `[0, π)`。
 - 角度基准（无向）：
  - HIFS：标准图像坐标角（0=东，π/2=南，π=西），无向（mod π）。
  - envs_new/tools：存储为 HIF 角（0=西，π/2=南，π=东），无向（mod π），由 `θ_hif=(π-θ_std) mod π` 映射得到。
  - 因均为“无向场（mod π）”，尽管存在 180° 相位差，但“线方向”语义等价，不构成冲突。
- Agent 朝向（cpp 环境）：以度为单位、图像坐标系定义：`0°=东, 90°=南, 180°=西, 270°=北`。
- v5 HIF 正则计算：`HIFCalculator` 将 HIF 弧度→度并做 +180° 相位对齐，随后最短环向差与无向归一（≤90°）。无论 HIF 来自 HIFS 还是 tools，方向差计算都正确、一致。

## 三、两套 HIF 生成管线：数据格式与角度定义
- 纹理提取（/home/lzh/HIFS）
  - `orientation_field_ui.py`
    - `angles = linspace(0, π, endpoint=False)`；掩码外置 `-1`；输出 `float32`。
    - 矢量可视化采用 `(u,v)=(cosθ, sinθ)`：θ=0→向右（东），θ=π/2→向下（南），θ=π→向左（西）。
    - 角差以 `min(|Δ|, π-|Δ|)` 实现无向（mod π）。
  - 保存：`save_results()` 输出 `orientation_map.npy`（`float32`，背景 `-1`，前景 `[0, π)`）。
- 形状/性质提取（envs_new/tools）
  - `envs_new/tools/generate_hif_from_field_shape.py`
    - 从 field mask 提取切线方向得到标准角 `θ_std`（0=东，π/2=南）。
    - 映射为 HIF 角：`θ_hif = (π - θ_std) mod π`，即 0=西、π/2=南、π=东（无向）。
    - 背景填 `-1.0`，保存至 `map_dir/hif/human_intent_field_{id}.npy`；类型 `float32`、前景 `[0, π)`。

## 四、区间内方向语义一致性
- 两者均为“无向场（mod π）”：θ 与 θ+π 表示同一条线方向；
- HIFS 的 0=东 与 tools 的 0=西 仅相位差 180°，在无向定义下语义等价；
- 因此：
  - 数据格式：一致；
  - 数值范围：一致；
  - 区间内每个值的“线方向”含义：一致（不因相位差而改变）。

## 五、v5 HIF 正则（方向差惩罚）正确性核查
- 位置：`envs_new/cpp_env_v5.py` → `HIFCalculator`
- 关键流程（要点复述）：
  - 取 `agent_direction`（度）与 `hif_map` 当前/上一步网格（弧度），权重 `0.3/0.7`；`hif < 0` 不计惩罚。
  - `_compute_angle_difference(agent_deg, hif_rad)`：
    - agent：度制正规化 `[0,360)`；
    - hif：弧度→度，做 `+180°` 相位对齐（将 HIF 映射到 Agent 坐标系）；
    - 角差取最短环向差（≤180°）；
    - 无向归一：若 `Δ>90°`，则 `Δ=180°-Δ`，得到 `Δ∈[0,90]`；
    - 返回 `Δ`（度）。
  - 奖励：`-coef*(0.3*Δ_cur + 0.7*Δ_last)`。
- 正确性分析：
  - 相位对齐与无向归一共同保证：无论 HIF 来源（HIFS 0=东 或 tools 0=西），差值计算均等价；
  - 单位一致性明确：Agent（度）、HIF（弧度）在函数内显式转换；
  - 网格索引一致：使用 `(y, x)` 访问 `hif_map[y, x]` 与 numpy 行列对应；
  - 背景处理：`hif < 0` 分支跳过，避免非场干扰。

## 六、潜在风险与建议
- 观测层插值：HIF 观测通道旋转采用线性插值，pad=-1.0 在边缘可能被平滑至 (-1,0)。这不影响奖励（奖励直接使用全局 hif_map），若需“观测中严格二值背景”，可对 HIF 通道单独用最近邻插值或增加 padding。
- 若未来切换“有向场”目标（区分 θ 与 θ+π）：需统一 HIF 角度基准（建议“标准角 0=东”），并移除“无向归一”，改用有向差（例如向量点积或有向环向差）。当前实现严格基于“无向”假设，与现有两套管线匹配。
- 场景对接：若直接用 HIFS 结果作为场景输入，建议保存为 `map_hif.npy` 或规范命名为 `hif/human_intent_field_{id}.npy`，并保证与对应 `field_{id}.png` 尺寸一致。

## 七、结论
- 两套 HIF 生成与 v5 环境消费端在“格式、范围、无向语义”上已对齐。数值相位差被 `HIFCalculator` 的相位对齐与无向归一完全吸收，不会引起方向正则误差。
- Cpp Agent 朝向、索引与单位处理与 HIF 正则计算匹配。
- 现有实现满足“纯覆盖 + 人类意图方向引导”的设计目标，无需代码修复；如需更严谨的观测显示或探索“有向场”，可按“建议”推进。
