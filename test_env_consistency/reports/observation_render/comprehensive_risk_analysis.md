## 二、Critical级问题详解

### 2.1 障碍物APF后处理缺失 🚨

**问题描述**：
新环境在`envs_new/cpp_env_v2.py`第152行缺少关键的后处理步骤

**代码对比**：
```python
# 旧环境 (envs/cpp_env_v2.py:64)
apf_obstacle = np.maximum(apf_obstacle, np.logical_and(self.map_obstacle, self.map_mist))

# 新环境 - 缺失此行！
```

**影响分析**：
- 障碍物边界的APF值可能为0
- Agent无法正确感知障碍物边缘
- 导致碰撞风险增加

**修复方案**：
```python
# 在envs_new/cpp_env_v2.py line 152后添加：
apf_obstacle = np.maximum(apf_obstacle, np.logical_and(map_obstacle, map_mist))
```

### 2.2 随机数生成器状态不一致 🚨

**问题描述**：
多个组件使用独立的随机数生成器，破坏了环境的确定性

**影响分析**：
- 相同种子无法复现相同轨迹
- 训练结果不可重复
- 调试困难度增加

**修复方案**：
```python
# 确保所有组件使用统一的RNG
self.observation_generator.set_random_generator(self.np_random)
self.renderer.set_random_generator(self.np_random)
```

## 三、High级问题分析

### 3.1 多尺度观测实现差异

**差异点**：
- 第4层处理：旧环境简单裁剪，新环境可能resize
- 中心计算：边界条件处理不同

**风险评估**：可能导致5-10%的观测差异

### 3.2 噪声应用机制不一致

**差异点**：
- 旧：直接修改agent坐标
- 新：生成噪声坐标，不修改原始状态

**风险评估**：可能导致细微的行为差异

### 3.3 第一人称视图提取差异

**差异点**：
- 提取方法完全不同
- 旋转中心计算方式有差异

**风险评估**：边界情况下可能有较大差异
